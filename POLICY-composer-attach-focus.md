# 정책 — Composer 첨부(+) 메뉴 & 자동 확대 포커스

> 대상: `OpenCodeShell` composer (participant.jsx) · C-2/C-3/C-4 공용 셸
> 최종 갱신: 2026-06-11

## 1. 배경

입력창(composer doc)은 **자동 확대(auto-expand)** 가 켜져 있으면
포커스 상태에서 커지고(big), blur 되면 작아진다(small).

```
isBig = autoExpand && focused
```

### 확대(big)의 정의 — 높이 + 폭
확대는 단순 **높이** 증가에 그치지 않는다. 확대 상태에서는 composer 가
**폭도 우측 미리보기 영역까지 침범하여 전체 폭**으로 확장된다(미리보기 위로 오버레이).

- 작은 상태: 폭 = 좌측 컬럼 폭 (`colW − 28`)
- 확대 상태: 폭 = 셸 전체 폭 (`shellW − 28`), `z-index` 로 미리보기 위 오버레이
- **대각선 성장**: 폭(composer `width`)과 높이(doc body `height`)를
  **동일한 duration·easing**(`var(--dur-base)` · `--ease-decelerate`)으로 묶어,
  좌우·상하를 따로 움직이지 않고 코너가 **직선 대각선**으로 자연스럽게 확대/축소되게 한다.
  (둘의 easing 이 다르면 코너 궤적이 휘어 부자연스러움 — 반드시 일치시킬 것.)
- 확대 시 그림자 강화(elevation)로 떠 있는 느낌 부여.
- 스플리터/높이 핸들 **드래그 중에는 transition 비활성**(즉시 추종).

첨부(+) 버튼·메뉴를 클릭하면 DOM 상 포커스가 입력창에서 버튼으로
옮겨가며 `focused=false` 가 되어 **의도치 않게 입력창이 접히는** 문제가 있었다.

## 2. 정책 (확정)

| 동작 | 입력창 포커스 | 결과 |
|------|--------------|------|
| **+ 버튼 클릭** (메뉴 열기/닫기) | **유지** | 클릭 시점 상태 보존 — 포커스였으면 확대 유지, 아니면 작은 상태 유지 |
| **메뉴 바깥 클릭** (닫기) | **유지** | 자동 확대 안 풀림 |
| **이미지 선택** | **유지** | 자동 확대 안 풀림 |
| **파일 선택** | **유지** | 자동 확대 안 풀림 |
| **캡처 선택** | **해제** | 화면 캡처를 위해 입력창 접힘 — **유일한 예외** |

### 핵심 원칙
- 첨부 상호작용은 기본적으로 **입력창 포커스 상태를 변경하지 않는다.**
- 단, **캡처**는 사용자가 화면 영역을 캡처해야 하므로 입력창을 일시적으로 접는다.
  → 이는 **캡처할 영역을 입력창이 가리지 않게** 하기 위함이며,
  캡처가 끝나면(이미지 삽입 시) 자동 확대 상태로 **복귀**한다. (§4 참조)

## 3. 구현 방식

- **포커스 유지**: 해당 요소 `onMouseDown={(e) => e.preventDefault()}`
  → 버튼이 포커스를 빼앗지 않아 입력창 `blur` 미발생 (깜빡임 없음).
  - 적용: + 버튼, 바깥 오버레이, 이미지·파일 메뉴 항목.
- **포커스 해제(캡처만)**: `onClick={() => { setFocused(false); setAttachOpen(false); }}`
  → `preventDefault` 없이 명시적으로 `focused=false` 처리.

## 4. 캡처 인터랙션 상세 (영역 캡처)

캡처 메뉴를 선택하면 다음 플로우가 진행된다:

1. **포인터 전환** — 마우스 포인터가 **캡처기(크로스헤어/캡처 커서)** 모양으로 바뀐다.
   동시에 입력창은 접혀(자동 축소) 캡처 대상 영역을 가리지 않는다.
2. **영역 드래그** — 화면의 특정 지점을 **클릭한 채 드래그**하면
   드래그 범위가 **사각형 선택 영역**으로 표시된다.
3. **캡처 확정 & 삽입** — 마우스 클릭을 떼면 해당 사각형 영역이 캡처되고,
   **입력창에서 마지막으로 포커싱돼 있던 커서 위치**에 캡처 이미지가 삽입된다.
4. **복귀** — 이미지 삽입 시 입력창은 다시 포커스되며,
   **자동 확대가 켜져 있으면 확대된 상태로 복귀**한다.

> 캡처를 누른 직후 입력창이 자동 축소되는 이유는 **캡처할 영역을 입력창이 가리지 않게**
> 하기 위함이며, 축소는 일시적이다 — 캡처 완료(삽입) 시 원래(확대) 상태로 돌아온다.

## 5. 미검증 / 후속

- 캡처 클릭 후 실제 영역 드래그·캡처·이미지 삽입 플로우는 **시안 미구현**(상태 전환만 정의).
- "마지막 포커싱 커서 위치" 보존 방식(selection range 기억)은 구현 시 별도 설계 필요.
- 캡처 도중 ESC 취소 시 입력창 복귀(자동 확대 복원) 정책은 §4-4 준용.
- 키보드 탐색(Tab/Esc) 시 포커스 정책은 별도 검토 필요.
- 메뉴 열린 채 스플리터 드래그·스크롤 시 fixed 팝오버 재배치 안 됨(열림 시점 좌표 고정).
