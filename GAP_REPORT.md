**모호함**: 참가자가 paused 화면에서 **직접 닫을 수 있는지** (예: X 버튼) 명세 없음. 2026-05-26 해소: `e6-paused`(구 c1-paused) 모달에 닫기·X 버튼 없음 — 참가자-로그인-기획 §"일시정지 — 작업 화면 위 오버레이"에 "참가자는 이 오버레이를 직접 닫을 수 없다" 명시 완료.

**조치 권장**: 기획문서에 명시 — "참가자는 paused 오버레이를 직접 닫을 수 없으며 운영자 재시작만이 해제 트리거다" (또는 디자인 의도를 명시).

**우선순위**: ★★

## 3. ⚠️ 양쪽 모두 수정 필요 (정합성 깨짐)

### 3-1. **[High]** "튜토리얼 건너뛰기" 트리거 UI 불일치
**기획문서** (`어드민-역할-기획.md` L221~222):
> "버튼 구성: 운영자 화면 버튼은 `[튜토리얼 시작]`과 `[해커톤 시작]` 두 가지다. `[해커톤 시작]`은 ① 튜토리얼 대기와 ④ 해커톤 대기 두 상태 모두에 나타나며 ... ① 튜토리얼 대기에서 `[해커톤 시작]`: 튜토리얼을 진행하지 않고 곧바로 ⑤로 간다"

**현 디자인** (operator.jsx tutorial_waiting):
- `[튜토리얼 시작]` 버튼 (primary)
- `"튜토리얼 건너뛰고 바로 시작"` **텍스트 링크** (별도 UI)

→ 기획문서는 [해커톤 시작] **버튼**으로 건너뛰기를 명시하나, 디자인은 별도 텍스트 링크로 분리

**판단**: 디자인 의도(별도 약한 링크 = 실수 방지)가 더 안전하므로 디자인 유지 권장. 기획문서를 디자인에 맞춰 갱신.

**조치 권장 (양방향)**:
- ✅ 와이어링: 텍스트 링크에 `data-action="skip"` 추가 완료 (이번 작업)
- 📝 기획문서: §"규칙" 버튼 구성 절을 "튜토리얼 시작 버튼 + 별도 약한 텍스트 링크로 건너뛰기 진입"으로 수정

**우선순위**: ★★★★

### 3-2. **[Medium]** 자동 전환 매트릭스의 운영자 액션 ↔ 모달 위계
**기획문서**: 자동 전환은 운영자가 "[튜토리얼 시작]을 누른 순간"부터 발동
**디자인 + 와이어링**: 운영자가 [튜토리얼 시작] 누름 → **확인 모달** (현재 누락) → 확정 → 자동 전환

**갭**: 기획문서엔 모달 단계가 명시되어 있지만(§규칙 L213), 자동 전환 매트릭스(§5 L235)엔 모달 단계 누락 → 두 문장이 서로 다른 흐름을 함의

**조치 권장 (양방향)**:
- 📝 기획문서: 자동 전환 매트릭스에 "확인 모달 후" 명시 ("[튜토리얼 시작] **확인 후** | 대기실 ① → C-2 셀프 튜토리얼")
- ✅ 와이어링: 현재 ACTIONS는 모달 누락 상태 그대로 — 누락 모달 추가 후 와이어링도 모달 경유로 변경

**우선순위**: ★★★

### 3-3. **[Medium]** 코드 입력 검증 명세 vs 디자인 에러 상태 매칭

**조치 권장 (양방향)**:

**우선순위**: ★★★
